Introdução aos Elementos de Estrutura

HTML5 revolucionou a forma como estruturamos documentos, trazendo elementos semânticos que dão significado ao conteúdo. Antes, tudo era feito com <div> e <span> genéricos. Hoje, temos tags específicas que comunicam a função de cada região da página tanto para navegadores quanto para tecnologias assistivas.

Bloco vs Inline: O Conceito Fundamental

ELEMENTO DE BLOCO
<div> e similares

Ocupa 100% da largura disponível, empilha verticalmente. Aceita margem/padding em todos os lados.

Bloco 1
Bloco 2
ELEMENTO INLINE
<span> e similares

Ocupa apenas o espaço do conteúdo, flui horizontalmente. Margem/padding vertical não afeta layout.

Inline 1 Inline 2 Inline 3
💡 Analogia: Pense em elementos de bloco como caixas empilhadas em uma prateleira (cada uma ocupa toda a largura), e elementos inline como palavras em uma frase (fluem lado a lado até acabar o espaço).

Elementos Genéricos: div e span

Embora o HTML5 tenha trazido alternativas semânticas, <div> e <span> ainda são essenciais quando não há um elemento semântico específico ou para agrupamento puramente visual/funcional.

Uso Correto de Divs e Spans 
<!-- DIV: Container de bloco genérico -->
<div class="card-container">
  <div class="card-header">
    <h2>Título do Card</h2>
  </div>
  <div class="card-body">
    <p>Conteúdo do card...</p>
  </div>
</div>

<!-- SPAN: Container inline genérico -->
<p>
  O preço é <span class="currency">R$</span>
  <span class="amount">99</span>,
  <span class="cents">90</span>
</p>

<!-- Destaque inline sem semântica específica -->
<p>
  Entregamos em <span style="color: #10b981; font-weight: bold;">24 horas</span>
  para São Paulo.
</p>
⚠️ Divitis: O Mal do Excesso

Evite o "síndrome da div" - usar <div> para tudo quando existem alternativas semânticas. Isso prejudica SEO e acessibilidade.

  • ❌ <div class="header"> → ✅ <header>
  • ❌ <div class="nav"> → ✅ <nav>
  • ❌ <div class="article"> → ✅ <article>

Quando Ainda Usar Div?

WRAPPERS
Containers de Layout

Para agrupar elementos apenas para estilização CSS (Flexbox/Grid containers) quando não há significado semântico.

COMPONENTES
Widgets Reutilizáveis

Cards, modais, tooltips e componentes de UI que são genéricos por natureza.

UTILITÁRIOS
Helpers de Espaçamento

Divs vazios para clearfix (legado), divisores decorativos ou espaçadores.

Elementos Semânticos de Estrutura

HTML5 introduziu elementos que descrevem o propósito do conteúdo, não apenas sua apresentação. Isso é crucial para leitores de tela, mecanismos de busca e manutenção de código.

CABEÇALHO
<header>

Introdução de conteúdo ou navegação. Pode haver múltiplos por página (ex: header de um artigo).

Header
Logo, título principal, navegação primária
CONTEÚDO PRINCIPAL
<main>

Conteúdo único e principal da página. Apenas um por documento, não pode ser filho de article/aside/nav.

Main
Conteúdo central, exclusivo desta página
SEÇÃO GENÉRICA
<section>

Agrupamento temático de conteúdo. Geralmente tem um heading (h2-h6) como primeiro filho.

Section
Capítulos, áreas temáticas, grupos de funcionalidade
CONTEÚDO INDEPENDENTE
<article>

Conteúdo auto-contido que faria sentido fora da página (post de blog, card de produto, comentário).

Article
Posts, produtos, widgets de conteúdo reusáveis
CONTEÚDO TANGENCIAL
<aside>

Conteúdo relacionado mas separado do fluxo principal (sidebars, boxes de informação, publicidade).

Aside
Sidebars, links relacionados, bio do autor
RODAPÉ
<footer>

Informações de fechamento: autor, copyright, links relacionados. Pode haver múltiplos.

Footer
Copyright, links de contato, mapa do site

Article vs Section: Entendendo a Diferença

<article> <section> Conteúdo independente (faz sentido sozinho) Agrupamento temático (precisa de contexto) Ex: Post de blog, comentário, produto Ex: Introdução, capítulo, área de features Pode conter sections dentro Pode conter articles dentro Syndicatable (pode ser distribuído via RSS) Estrutural (organiza o documento)
Aninhamento Correto 
<!-- Estrutura de Blog: Section contém Articles -->
<section id="ultimos-posts">
  <h2>Últimos Artigos</h2>
  
  <article>
    <header>
      <h3>Título do Post</h3>
      <time datetime="2024-03-20">20 de Março</time>
    </header>
    <p>Resumo do conteúdo...</p>
    <footer>
      <p>Por João Silva</p>
    </footer>
  </article>
  
  <article>
    <!-- Outro post -->
  </article>
</section>

<!-- Article contendo Sections -->
<article>
  <h1>Guia Completo HTML</h1>
  
  <section>
    <h2>Introdução</h2>
    <p>...</p>
  </section>
  
  <section>
    <h2>Conceitos Avançados</h2>
    <p>...</p>
  </section>
</article>

Landmarks e Estrutura de Página

Landmarks são regiões da página que ajudam tecnologias assistivas (leitores de tela) a navegar rapidamente. Elementos semânticos HTML5 mapeiam automaticamente para landmarks ARIA.

Anatomia de uma Página Semântica

Header
banner landmark: Define o cabeçalho do site, geralmente contendo o logotipo, nome da empresa e, às vezes, a navegação principal (logo + menu).
Navigation
navigation landmark: Define uma seção contendo links de navegação, como o menu principal, menu de rodapé ou navegação interna da página.
Article 1
Section
Section
Aside
complementary landmark: é um elemento de acessibilidade web que define uma seção de suporte, como uma barra lateral, lateral ou caixa de destaque, que complementa o conteúdo principal
Footer
contentinfo landmark: é uma região que identifica informações de rodapé comuns em um site, como direitos autorais (copyright), links de privacidade e acessibilidade
🎯 Navegação por Landmarks: Usuários de leitores de tela (NVDA, JAWS, VoiceOver) podem pressionar teclas de atalho para pular diretamente para header, nav, main, aside ou footer. Sem essas tags, a navegação fica linear e lenta.

Elemento <nav>: Navegação Estratégica

Uso Correto do Nav 
<!-- Navegação principal -->
<nav aria-label="Navegação principal">
  <ul>
    <li><a href="/" aria-current="page">Home</a></li>
    <li><a href="/sobre">Sobre</a></li>
    <li><a href="/contato">Contato</a></li>
  </ul>
</nav>

<!-- Múltiplos navs precisam de labels -->
<nav aria-label="Redes sociais">
  <ul>
    <li><a href="#">Twitter</a></li>
    <li><a href="#">LinkedIn</a></li>
  </ul>
</nav>

<!-- Breadcrumb (caminho de navegação) -->
<nav aria-label="Breadcrumb">
  <ol>
    <li><a href="/">Home</a></li>
    <li><a href="/produtos">Produtos</a></li>
    <li aria-current="page">Notebooks</li>
  </ol>
</nav>
⚠️ Nem toda lista de links precisa ser <nav>: Use <nav> apenas para blocos de links de navegação principais. Links no footer (copyright, políticas secundárias) ou em textos inline não precisam de nav.

Details e Summary: Conteúdo Colapsável

Os elementos <details> e <summary> criam widgets de "acordeão" nativos, sem necessidade de JavaScript. Perfeitos para FAQs, termos de uso e conteúdo opcional.

Acordeão Sem JavaScript 
<details>
  <summary>Quais formas de pagamento são aceitas?</summary>
  <p>Aceitamos cartões de crédito (Visa, Master, Amex), 
  Pix e boleto bancário. Pagamentos via PayPal estão 
  disponíveis para compras internacionais.</p>
</details>

<details open>  <!-- Aberto por padrão -->
  <summary>Política de Devolução</summary>
  <ul>
    <li>7 dias para arrependimento</li>
    <li>Produto deve estar lacrado</li>
    <li>Frete por conta do comprador</li>
  </ul>
</details>

<!-- Aninhado (Accordion multinível) -->
<details>
  <summary>Categoria: Eletrônicos</summary>
  <details style="margin-left: 20px; margin-top: 10px;">
    <summary>Computadores</summary>
    <ul>
      <li>Notebooks</li>
      <li>Desktops</li>
    </ul>
  </details>
</details>
Demonstração Interativa
Clique para expandir (exemplo fechado)
Este conteúdo estava oculto! Perfeito para manter a página limpa e mostrar informações apenas quando necessário.
Este já vem aberto (atributo open)
Útil quando a informação é importante mas opcional, ou quando queremos mostrar o estado por padrão.
💡 Estilização Avançada: O elemento <summary> aceita ::marker para customizar a seta (usando list-style) ou pode ser escondida para criar interfaces completamente customizadas com CSS.

Figure e Figcaption: Conteúdo Autocontido

<figure> representa conteúdo autocontido frequentemente com uma legenda (<figcaption>). Pode conter imagens, diagramas, código, poemas ou citações.

Uso com Imagem e Código 
<!-- Figura com imagem -->
<figure>
  <img src="grafico-vendas.jpg" alt="Gráfico de crescimento trimestral">
  <figcaption>
    Figura 1: Crescimento de 150% no trimestre Q4-2024.
    Fonte: Departamento de Analytics.
  </figcaption>
</figure>

<!-- Figura com bloco de código -->
<figure>
  <pre><code>function hello() {
  console.log("Hello World");
}</code></pre>
  <figcaption>
    Listagem 2.1: Função de saudação básica em JavaScript
  </figcaption>
</figure>

<!-- Múltiplas imagens em uma figura -->
<figure>
  <img src="foto1.jpg" alt="Vista frontal do produto">
  <img src="foto2.jpg" alt="Vista lateral do produto">
  <figcaption>
    Produto visto de diferentes ângulos. Clique para ampliar.
  </figcaption>
</figure>
📊 Quando usar Figure vs Div/Article:
  • Use <figure> quando o conteúdo é referenciado no texto principal ("como mostra a Figura 1...")
  • Use <figure> quando o conteúdo pode ser removido sem afetar o fluxo principal do documento
  • Não use apenas para posicionar imagens decorativas - use <div> nesse caso

Acessibilidade em Layouts

Ordem de Leitura Linear

⚠️ Problema Comum: CSS Grid e Flexbox permitem reordenar visualmente elementos com order. Isso pode criar uma experiência confusa para usuários de teclado se a ordem visual não corresponder à ordem do DOM.
Estrutura Acessível 
<!-- ✅ BOM: Ordem lógica no DOM -->
<main>
  <article>Conteúdo principal</article>
  <aside>Conteúdo complementar</aside>
</main>

<style>
/* Visualmente side-by-side, mas mantém ordem semântica */
.container {
  display: grid;
  grid-template-columns: 2fr 1fr;
  gap: 30px;
}
</style>

<!-- ❌ EVITE: Reordenação que confunde -->
<main style="display: flex;">
  <aside style="order: 1;">Sidebar</aside>  <!-- Aparece primeiro visualmente -->
  <article style="order: 2;">Conteúdo</article> <!-- Mas está depois no DOM -->
</main>

Regiões e Navegação por Teclado

✓
Use <main> para que usuários possam pular direto ao conteúdo
✓
Adicione aria-label em múltiplos <nav> ou <aside>
✓
Use aria-labelledby para conectar headings às sections
✓
Evite níveis excessivos de aninhamento (máximo recomendado: 3-4 níveis)
Landmarks com ARIA Labels 
<!-- Navegação com identificação clara -->
<nav aria-label="Principal">...</nav>
<nav aria-label="Redes sociais">...</nav>

<!-- Seção identificada pelo título -->
<section aria-labelledby="secao-contato">
  <h2 id="secao-contato">Entre em Contato</h2>
  <!-- Leitores de tela anunciam: "Seção, Entre em Contato" -->
</section>

<!-- Complementary regions precisam de label quando há múltiplas -->
<aside aria-label="Posts relacionados">...</aside>
<aside aria-label="Newsletter">...</aside>
🛠️ Ferramenta de Teste: Use o Accessibility Inspector do Chrome DevTools ou o plugin axe DevTools para verificar se todas as regiões da sua página estão corretamente identificadas como landmarks.

🎯 Exercícios Práticos:
  1. Reescreva um site antigo que use apenas <div>, substituindo por tags semânticas apropriadas
  2. Crie 3 layouts diferentes (blog, dashboard, landing) usando apenas HTML semântico (sem CSS)
  3. Teste seus layouts com um leitor de tela (NVDA gratuito para Windows, VoiceOver no Mac)
  4. Valide se sua estrutura de headings está correta (h1 → h2 → h3, sem pular níveis)
  5. Implemente um sistema de "skip links" funcional em um template multi-seção
🔗 Integração com CSS: Após dominar a estrutura HTML semântica, estude CSS Grid e Flexbox para posicionar esses elementos. A combinação de HTML5 semântico + CSS moderno é a base de qualquer interface profissional atual.
